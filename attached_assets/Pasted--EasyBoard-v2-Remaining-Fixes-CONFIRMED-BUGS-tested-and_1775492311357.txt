# EasyBoard v2 — Remaining Fixes

## CONFIRMED BUGS (tested and broken)

### 1. Vote title "Untitled Resolution" on AI-created votes
AI proposes "Create Vote" from a committee submission → Secretary approves → vote created with title "Untitled Resolution" instead of something descriptive.

**Fix:** In the create_vote handler in `pendingActions.ts`, extract the title from `action_data.description` or the first sentence of the resolution text. Fallback to "Resolution — [board abbreviation]" if nothing available.

**Test:** Upload investment proposal → approve Create Vote → vote title should be descriptive (e.g., "Approval of Green Hydrogen Feasibility Study Budget"), not "Untitled Resolution."

---

### 2. Pending member names not resolved in vote detail
Secretary vote detail shows "Pending member" with "?" avatar for members who haven't voted. Only members who already voted show their real names. The Secretary needs to see WHO hasn't voted.

**Fix:** The Member Votes query must join `board_memberships` → `people` for the vote's board to get ALL members, not just those with `vote_records`. Show each member's full name and title, with "Not yet voted" for those without a record.

**Test:** Open vote detail → all 6 SIC members should show by name. Sarah Chen shows "Approved", the other 5 show their real names with "Not yet voted."

---

### 3. Observer can't see supporting documents on votes
Board member sees "1 Supporting Document" on the vote card. Observer sees the same vote but no supporting documents mention.

**Fix:** Observer board room vote card should display the same supporting document count and download links as the board member view. Observers need read-only access to materials.

**Test:** Login as d.park (observer) → SIC room → vote card → must show "1 Supporting Document."

---

### 4. Observer dashboard "Open Votes" counter shows 0
Observer dashboard displays "OPEN VOTES: 0" even though the SIC board room correctly shows an open vote.

**Fix:** Dashboard counter query for observers should include votes on boards where the user is an observer member, not just voting members.

**Test:** Login as d.park → dashboard → "OPEN VOTES" should show 1.

---

### 5. Custom approval rules incomplete
The "Custom" option exists with Min Approvals and Quorum fields. But governance requires more:

**Missing features — build these:**

**a) Recused members** — Multi-select dropdown of board members. When recused: excluded from voting (no vote buttons shown to them), excluded from total voter count and quorum calculation. Essential for conflict-of-interest governance.

**b) Key approvers** — Multi-select of specific members whose individual "Approved" vote auto-approves the resolution regardless of total count. Example: "If the Chairman votes Approved, the resolution passes immediately." This is used for delegated authority and emergency decisions.

**c) Deadline behavior** — Radio buttons:
  - "Lapse" — vote expires with no result if deadline passes without quorum
  - "Extend" — auto-extend by 7 days if quorum not reached at deadline
  - "Notify" — alert Secretary 48 hours before deadline

**d) Plain-English summary** — Display below the configuration. Example: "Requires at least 5 approvals from 9 eligible voters (James O'Brien recused). If Nadia Petrov approves, resolution passes immediately. Quorum: 8 must vote. Vote lapses if deadline passes without quorum."

**Store as JSON** in the votes table approval_rule column. The vote result calculation logic must handle: recused exclusion, key approver auto-pass, quorum validation, deadline behavior.

**Test:**
1. Create Custom vote: min 3 approvals, quorum 4, O'Brien recused, Petrov as key approver
2. O'Brien enters board room → should NOT see vote buttons
3. Petrov votes Approved → vote auto-closes as Approved (key approver rule)
4. Create another Custom vote without key approvers → 3 members vote, but only 3 of 4 quorum met → vote should NOT resolve

### 6. Extend deadline timezone display
Extending deadline to 2026-05-15 displays as "15 May 2026, 04:00 am" instead of just "15 May 2026."

**Fix:** When deadline is date-only (no time component), display date without time. Or store as start-of-day in local timezone.

---

### 7. Minutes content placeholder on new AI approvals
When Secretary uploads a NEW document and approves "Create Minutes," the content sometimes shows "[Minutes content — please review and edit before finalizing]" instead of the actual document text. Pre-seeded minutes have real content, but freshly AI-approved minutes from new uploads may not.

**Fix:** Verify the create_minutes handler in `pendingActions.ts` always reads the source document file content when `documentId` is present, regardless of what `action_data.content` contains. The file content should always take priority over any AI-generated placeholder.

**Test:** Upload a new .txt file with board minutes → approve Create Minutes → open the new minutes → content must be the uploaded document text.
