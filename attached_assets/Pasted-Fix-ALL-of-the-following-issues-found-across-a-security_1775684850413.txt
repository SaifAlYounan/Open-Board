## 2 — Add UUID validation to boards routes

**File:** `artifacts/api-server/src/routes/boards.ts`

**Problem:** GET /boards/:invalid-uuid returns 500 (server crash). All other routes (votes, meetings, minutes) correctly return 400. Boards is missing the validation.

**Fix:** Add at the top of the boards router:
```typescript
const UUID_REGEX = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;
router.param("id", (_req, res, next, id) => {
  if (!UUID_REGEX.test(id)) {
    res.status(400).json({ error: "Invalid id format" });
    return;
  }
  next();
});
```

## 3 — Add pagination caps to all list endpoints

**Files:** All route files with list/GET endpoints

**Problem:** List endpoints return ALL records with no server-side limit. A client can request `?limit=999999`.

**Fix:** Add to every list endpoint:
```typescript
const MAX_LIMIT = 200;
const limit = Math.min(parseInt(req.query.limit as string) || 50, MAX_LIMIT);
const offset = Math.max(parseInt(req.query.offset as string) || 0, 0);
```

Apply to: boards, votes, meetings, minutes, documents, tasks, people, audit, dashboard endpoints. Use `.limit(limit).offset(offset)` on all queries.

## 9 — Fix XSS sanitization stripping entire content

**File:** `artifacts/api-server/src/lib/sanitize.ts`

**Problem:** `sanitizeText("<script>alert(1)</script>")` returns `""` (empty string). It strips the tags AND the text content inside them. If someone types `Test <b>bold</b> text`, it should keep "Test bold text".

**Fix:** Update sanitizeText to preserve text content:
```typescript
import sanitizeHtml from "sanitize-html";

export function sanitizeText(input: string): string {
  return sanitizeHtml(input, {
    allowedTags: [],
    allowedAttributes: {},
    textFilter: (text) => text, // preserve text content
  });
}
```

Test: `sanitizeText("<script>alert(1)</script>")` should return `"alert(1)"` (text preserved, tag stripped).
Test: `sanitizeText("Board Meeting <b>Q1</b>")` should return `"Board Meeting Q1"`.

## 11 — Fix logout to clear cookie properly

**File:** `artifacts/api-server/src/routes/auth.ts`

**Problem:** Logout clears the cookie but the JWT remains valid server-side. While full server-side token blacklisting is complex, the cookie clearing should at least be consistent.

**Fix:** Ensure the logout endpoint clears the cookie with the SAME attributes it was set with:
```typescript
router.post("/logout", (req, res) => {
  res.clearCookie("token", {
    httpOnly: true,
    secure: process.env.NODE_ENV === "production",
    sameSite: "strict",
    path: "/",
  });
  res.json({ message: "Logged out" });
});
```

The `path` must match the original cookie path, otherwise browsers won't clear it.
