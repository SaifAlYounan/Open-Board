## Fix 1 — JWT: Move from localStorage to HttpOnly cookies (MEDIUM)

The app stores JWT tokens in localStorage, which any malicious script on the page can read and steal.

**Backend changes (artifacts/api-server/src/routes/auth.ts or wherever login response is):**

After generating the JWT token, instead of sending it in the response body, set it as an HttpOnly cookie:

```typescript
res.cookie("token", token, {
  httpOnly: true,
  secure: true,
  sameSite: "strict",
  maxAge: 24 * 60 * 60 * 1000 // 24 hours, adjust to match your JWT expiry
});
res.json({ success: true }); // don't include token in body
```

**Backend changes (auth middleware):**

Update `requireAuth` middleware to read the token from the cookie instead of (or in addition to) the Authorization header:

```typescript
const token = req.cookies?.token || req.headers.authorization?.split(" ")[1];
```

Add cookie-parser if not already installed:
```
npm install cookie-parser
```

And in app.ts:
```typescript
import cookieParser from "cookie-parser";
app.use(cookieParser());
```

**Frontend changes:**

Remove all `localStorage.getItem("token")` and `localStorage.setItem("token", ...)` calls. The browser sends the cookie automatically with every request. Make sure all API calls use `credentials: "include"` in fetch options.

**Logout:** Clear the cookie server-side with `res.clearCookie("token")`.

## Fix 3 — Add request body size limit (LOW)

**File:** `artifacts/api-server/src/app.ts`

Find `express.json()` and `express.urlencoded()` and add limits:

```typescript
app.use(express.json({ limit: "1mb" }));
app.use(express.urlencoded({ extended: true, limit: "1mb" }));
```

This does NOT affect file uploads — those go through multer which already has its own limits (20MB for votes, 10MB for tasks).
