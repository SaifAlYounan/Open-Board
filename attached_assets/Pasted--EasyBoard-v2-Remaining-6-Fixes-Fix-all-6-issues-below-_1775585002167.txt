## FIX 1 — close_task closes wrong task (CRITICAL)

**File:** `artifacts/api-server/src/routes/pendingActions.ts`

**Current code (broken):**
```typescript
const [t] = await db.select().from(tasksTable);
```
This selects ALL tasks and takes the first one. It ignores `task_number` entirely.

**Fix — replace with:**
```typescript
const [t] = await db.select().from(tasksTable)
  .where(eq(tasksTable.taskNumber, task_number))
  .limit(1);
```

Make sure `eq` is imported from `drizzle-orm`. If `task_number` is not provided but `taskId` is, use `taskId` directly (which is the existing first check — keep that).

## FIX 2 — CORS wildcard

**File:** `artifacts/api-server/src/app.ts`

**Current code:**
```typescript
cors: { origin: "*", methods: ["GET", "POST"] }
```

**Fix — replace with:**
```typescript
cors: {
  origin: process.env.ALLOWED_ORIGIN || true,
  methods: ["GET", "POST", "PATCH", "DELETE"],
  credentials: true
}
```

`true` allows the request origin (same as wildcard but with credentials support). For production, set `ALLOWED_ORIGIN` env var to the actual domain. Also add PATCH and DELETE to allowed methods since the app uses them.

## FIX 3 — Rate limiting on AI endpoints

**Install:** Add `express-rate-limit` to dependencies:
```bash
pnpm add express-rate-limit --filter @workspace/api-server
```

**File:** `artifacts/api-server/src/routes/ai.ts`

Add at the top:
```typescript
import rateLimit from 'express-rate-limit';

const aiLimiter = rateLimit({
  windowMs: 60 * 1000,  // 1 minute
  max: 10,              // 10 AI calls per minute per IP
  message: { error: 'Too many AI requests. Try again in a moment.' }
});
```

Apply to all AI routes:
```typescript
router.post('/classify', requireAuth, requireAdmin, aiLimiter, async (req, res) => { ... });
router.post('/command', requireAuth, requireAdmin, aiLimiter, async (req, res) => { ... });
router.post('/search', requireAuth, aiLimiter, async (req, res) => { ... });
```

Also add a stricter limiter on the auth endpoint:
```typescript
const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,  // 15 minutes
  max: 10,                    // 10 login attempts per 15 min
  message: { error: 'Too many login attempts.' }
});
```
Apply to the login route.
