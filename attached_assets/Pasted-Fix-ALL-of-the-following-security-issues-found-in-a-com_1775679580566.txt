## CRITICAL FIXES

### C1 — Authenticate WebSocket connections

**File:** `artifacts/api-server/src/index.ts`

**Problem:** Socket.IO accepts connections with NO authentication. Anyone can connect and join any board/vote/minutes room to monitor live governance events.

**Fix:** Add authentication middleware to Socket.IO:

```typescript
import { verifyToken } from "./lib/auth"; // or wherever your JWT verify function is
import cookieParser from "cookie-parser";

// Parse cookies from the WebSocket handshake
io.use(async (socket, next) => {
  try {
    // Parse cookie from handshake headers
    const cookieHeader = socket.handshake.headers.cookie || "";
    const cookies = Object.fromEntries(
      cookieHeader.split(";").map(c => {
        const [key, ...val] = c.trim().split("=");
        return [key, val.join("=")];
      })
    );
    const token = cookies.token;
    if (!token) {
      return next(new Error("Authentication required"));
    }
    const user = verifyToken(token);
    if (!user) {
      return next(new Error("Invalid token"));
    }
    socket.data.user = user;
    next();
  } catch (err) {
    next(new Error("Authentication failed"));
  }
});
```

Then in the `join:board`, `join:vote`, `join:minutes` handlers, verify that `socket.data.user` has access to that entity before allowing the join. If the user doesn't have access, ignore the join request.

### C2 — Add access control to board detail endpoints

**File:** `artifacts/api-server/src/routes/boards.ts`

**Problem:** `GET /boards/:id` and `GET /boards/:id/members` return data to ANY authenticated user. Any user can read any board's membership.

**Fix:** Before returning board data, check that the requesting user is either:
- An admin, OR
- A member of that board (check the accessControl table)

If neither, return 403.

---

### C3 — Add access control to vote endpoints

**File:** `artifacts/api-server/src/routes/votes.ts`

**Problem:** `GET /votes/:id`, `GET /votes/:id/documents`, `GET /votes/:id/documents/:docId/download`, and `GET /votes/:id/certificate` have no access control.

**Fix:** For each of these endpoints, look up which board the vote belongs to, then check that the requesting user has access to that board. If not, return 403.

The certificate endpoint is especially sensitive — it shows how each board member voted.

### C4 — Fail-fast on missing seed password

**File:** `artifacts/api-server/src/seed.ts`

**Problem:** Falls back to `Meridian2024!` if `SEED_PASSWORD` env var is missing.

**Fix:** Change:
```typescript
const PASSWORD = process.env.SEED_PASSWORD || "Meridian2024!";
```
To:
```typescript
const PASSWORD = process.env.SEED_PASSWORD;
if (!PASSWORD) {
  throw new Error("SEED_PASSWORD environment variable is required. Set it in Replit Secrets before running seed.");
}
```

Then set `SEED_PASSWORD` in the Replit Secrets tab.
