# CRITICAL / HIGH FIXES (do these first)

## 1 — System reset requires admin + transaction

**File:** `artifacts/api-server/src/routes/system.ts`

**Problem:** The `POST /system/reset-data` endpoint uses `requireAuth` instead of `requireAdmin` — any logged-in user can wipe the database. Also, the multiple delete operations have no transaction wrapper, so a failure midway corrupts data.

**Fix:**
- Change middleware from `requireAuth` to `requireAuth, requireAdmin`
- Wrap all deletes in a database transaction:
```typescript
await db.transaction(async (tx) => {
  await tx.delete(minutesSignaturesTable);
  // ... all other deletes in order ...
});
```

## 4 — Rate limiting on write endpoints

**File:** `artifacts/api-server/src/routes/votes.ts`, `meetings.ts`, `minutes.ts`, `tasks.ts`, `documents.ts`

**Problem:** Only login and AI endpoints have rate limiting. All write operations (create vote, meeting, task, upload document) can be spammed.

**Fix:** Add a write rate limiter:
```typescript
const writeLimiter = rateLimit({
  windowMs: 60 * 1000,
  max: 30,
  message: { error: "Too many requests. Please wait." },
});
```

## 5 — UUID format validation on all ID parameters

**Files:** All route files that use `:id` params

**Problem:** Route parameters like `boardId`, `voteId`, `meetingId` are used directly without checking they're valid UUIDs.

**Fix:** Create a validation middleware or helper:
```typescript
const UUID_REGEX = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;

function validateUUID(param: string) {
  return (req, res, next) => {
    if (!UUID_REGEX.test(req.params[param])) {
      return res.status(400).json({ error: `Invalid ${param} format` });
    }
    next();
  };
}
```
Apply to all routes with `:id` parameters.
